# 🔔 通知与交互

弹幕服务内置了通知与交互功能，支持通过 Telegram Bot 和 Server酱³ Bot 接收系统通知、任务状态更新等消息。

> 此功能自 v2.5.3 起内置支持，灵感来源于 [表哥 bot](https://github.com/balge/misaka-danmuku-bot)

![通知与交互-添加交互渠道](/screenshot/通知与交互-添加交互渠道.png)

---

## Telegram Bot（内置）

v2.5.3 起，弹幕服务内置了 Telegram Bot 通知功能，无需额外部署即可使用。

### 功能特性

- 📢 接收系统通知和任务状态更新
- 🔄 接收弹幕源更新提醒
- ⚠️ 接收异常告警信息
- 🔍 通过 Bot 与弹幕服务进行基本交互

### 配置方法

1. 在 Telegram 中通过 [@BotFather](https://t.me/BotFather) 创建一个新的 Bot，获取 Bot Token
2. 获取您的 Telegram Chat ID（可通过 [@userinfobot](https://t.me/userinfobot) 获取）
3. 在 Web UI 中进入 **"设置"** → **"通知与交互"**
4. 选择 **Telegram Bot** 通知渠道
5. 填入 Bot Token 和 Chat ID
6. 点击测试按钮验证配置
7. 保存配置

::: tip 提示
如果您的服务器无法直接访问 Telegram API，需要先在 [代理配置](/config/设置#代理配置) 中设置代理。
:::

## Server酱³ Bot（内置）

Server酱³ 是一个轻量级的消息推送服务，支持通过微信、企业微信等渠道接收通知。目前为内测功能。

### 功能特性

- 📢 接收系统通知和任务状态更新
- 🔄 接收弹幕源更新提醒
- ⚠️ 接收异常告警信息

### 配置方法

Server酱³ Bot 使用的是 Bot 的 Key 和用户 UID 进行消息推送，而非传统 Server酱的 SendKey。

1. 前往 [Server酱³ 官网](https://sct.ftqq.com/) 注册账号
2. 获取您的 **Bot Key**（Bot 密钥）和 **用户 UID**
3. 在 Web UI 中进入 **"设置"** → **"通知与交互"**
4. 选择 **Server酱³** 通知渠道
5. 填入 Bot Key 和用户 UID
6. 点击测试按钮验证配置
7. 保存配置

::: tip Bot Key 与用户 UID
- **Bot Key**：在 Server酱³ 中创建 Bot 后获取的密钥，用于标识发送消息的 Bot
- **用户 UID**：您在 Server酱³ 中的用户唯一标识，用于指定消息接收人
:::

## 第三方方案：表哥 Telegram Bot

除了内置的通知功能外，[balge（表哥）](https://github.com/balge) 开发了一个功能更丰富的第三方 Telegram Bot，支持通过聊天界面直接管理弹幕服务器。

::: info 版本要求
弹幕库版本需要大于 v2.0.4 才可以使用此第三方 Bot。
:::

**项目地址**: [misaka-danmuku-bot](https://github.com/balge/misaka-danmuku-bot)

### 功能特性

相比内置通知，表哥 Bot 提供了更完整的交互管理能力：

- 🔍 搜索和导入新的影视作品
- 📚 管理媒体库、数据源和分集
- 📋 查看和管理后台任务
- 📊 查看系统状态
- 💬 通过聊天指令直接操作弹幕服务

### 使用方法

此 Bot 需要独立部署，请参考 [misaka-danmuku-bot](https://github.com/balge/misaka-danmuku-bot) 项目的 README 进行配置和使用。
